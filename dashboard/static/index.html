How it works

Each simulated driver is an agent defined by a usage profile. The simulator steps through one week at minute resolution, draining the battery while driving and recharging when plugged in. There are two ways to build your profile list:

  • Generate Profiles — sample N agents from the UK archetype distribution (weighted by real-world data). Each agent gets a small random offset on their plug-in schedule for realistic variation.
  • Add Profile Manually — define a custom driver by filling in each parameter. Useful for testing specific archetypes or edge cases.

Once you're happy with the active profiles, click Run Simulation. Results are written as parquet files to data_store/simulation_results/.
